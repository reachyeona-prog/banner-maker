# 카드 뉴스 구조 가이드

## 1. 카드 뉴스 한 세트의 표준 구성

| 순서 | 슬라이드 | 역할 |
| --- | --- | --- |
| 1장 | **표지 (E1)** | 후킹 — 한 번에 클릭하게 만드는 카피 |
| 2~6장 | **내지 (E2)** | 본론 — 데이터·맥락·결론을 흐름 있게 전개 |
| (선택) 마지막 | **클로징 내지** | CTA — 블로그 링크 / 다음 콘텐츠 안내 |

**총 4~7장**이 일반적. 5장이 가장 흔합니다. **8장 이상은 사용자가 끝까지 안 봅니다** — 욕심 부리지 마세요.

## 2. 흐름 설계 — "기-승-전-결" 모델

### (1) 표지 = 기 (Hook)
**한 줄 후킹 + pill 태그**. 사용자가 0.3초 안에 "흥미롭다, 한번 본다"를 결정.

- 질문형: "벚꽃 상권 매출, 어디가 얼마나 올랐을까?"
- `.zip` 압축형: "고유가 피해지원금 핵심 요약.zip"
- 데이터형: "외국인 결제, 이제는 일식이 1위?"

### (2) 첫 내지 = 승 (Context)
**왜 이 주제를 다루는지 맥락**. 데이터 출처·기간·범위·트리거 이벤트 등.

```
"벚꽃 나들이가 소비로 이어지며
{서울 주요 벚꽃 상권 매출}이 올랐어요!"
```

### (3) 중간 내지 = 전 (Detail)
**구체 데이터·인사이트**. 가장 중요한 슬라이드. 슬라이드 하나에 한 메시지·한 숫자.

```
"가장 큰 증가세를 보인 잠실 · 종로,
{어떤 업종 매출}이 늘었을까요?"
```

```
"{잠실}은 {일식 매출}이
전월 대비 {5.4배} 늘었어요"
```

### (4) 마지막 내지 = 결 (Conclusion / CTA)
**요약 한 줄** 또는 **CTA**. 너무 무겁지 않게.

```
"이런 트렌드,
우리 매장에도 적용해 보세요"
```

```
"자세한 데이터는
{페이히어 블로그}에서 만나요"
```

## 3. 표지 (E1) 작성법

### 구성 요소
- `pill` — 카테고리 라벨. 1~6자.
- `main1` — 메인 카피 첫 줄. 후킹의 도입.
- `main2` — 메인 카피 둘째 줄. 후킹의 클라이맥스.

### pill 라벨 예시 (페이히어 자주 쓰는 톤)

| pill | 콘텐츠 성격 |
| --- | --- |
| `NEWS` | 트렌드·시사 데이터 |
| `트렌드` | 결제·소비 트렌드 분석 |
| `데이터` | 페이히어 자체 결제 데이터 인사이트 |
| `매장 운영 TIP` | 사장님 운영 노하우 |
| `정책 가이드` | 지원금·세무·인허가 정보 |
| `사장님 사례` | 케이스 스터디 |
| `시즌 리포트` | 계절·이슈성 콘텐츠 |

### main1 → main2 연결

표지 카피는 **두 줄이 한 호흡**이에요. 끊김보다 흐름이 중요합니다.

**Good — 콤마로 호흡 + 질문**
```
main1: "벚꽃 상권 매출,"
main2: "어디가 얼마나 올랐을까?"
```

**Good — 명사구 + 압축**
```
main1: "고유가 피해지원금"
main2: "핵심 요약.zip"
```

**Bad — 두 줄이 다른 메시지**
```
main1: "벚꽃 상권 매출"
main2: "오늘은 결제 트렌드를 살펴봐요"  ← main1과 끊김
```

**Bad — 한 줄로 다 들어갈 내용을 억지 분할**
```
main1: "벚꽃"
main2: "상권 매출 분석"  ← 1줄로 충분
```

## 4. 내지 (E2) 작성법

### 구성 요소
- `main1` — 첫 줄. 맥락·전환·도입.
- `main2` — 둘째 줄. 결론·답·핵심 데이터.

### 흐름 패턴

**(A) 맥락 → 결론**
```
main1: "벚꽃 나들이가 소비로 이어지며"
main2: "{서울 주요 벚꽃 상권 매출}이 올랐어요!"
```

**(B) 전환 → 질문** — 다음 슬라이드로 시선 이동
```
main1: "가장 큰 증가세를 보인 잠실 · 종로,"
main2: "{어떤 업종 매출}이 늘었을까요?"
```

**(C) 데이터 노출**
```
main1: "{잠실}은 일식 매출이"
main2: "전월 대비 {5.4배} 늘었어요"
```

**(D) 비교·대조**
```
main1: "주중 매출은 {1.2배},"
main2: "주말 매출은 {2.8배} 뛰었어요"
```

### 슬라이드 간 일관성

- **시점·주체 일치**: 첫 슬라이드가 "사장님이 ~"이면 끝까지 사장님 주체로.
- **데이터 단위 일치**: 한 카드 뉴스 안에서 "전년 대비" / "전월 대비" 섞지 말 것.
- **강조 컬러 의미 통일**: `{}` 키워드는 슬라이드마다 비슷한 결(숫자만, 또는 지역명만)로 묶이면 깔끔.

## 5. 분량별 흐름 예시

### 4장 구성 (간결형)
1. 표지 — 후킹
2. 내지 — 맥락 + 결론
3. 내지 — 데이터 1
4. 내지 — 시사점 (또는 CTA)

### 5장 구성 (표준)
1. 표지 — 후킹
2. 내지 — 맥락
3. 내지 — 데이터 1
4. 내지 — 데이터 2 (비교/대조)
5. 내지 — 결론/시사점/CTA

### 6장 구성 (데이터 풍부)
1. 표지 — 후킹
2. 내지 — 맥락
3. 내지 — 핵심 데이터 (지역 또는 업종)
4. 내지 — 비교 데이터 1
5. 내지 — 비교 데이터 2
6. 내지 — 시사점/CTA

## 6. 흔한 실수 회피

| 실수 | 예시 | 수정 |
| --- | --- | --- |
| 표지가 너무 평이 | "벚꽃 상권 매출 변화" | "벚꽃 상권 매출, 어디가 얼마나 올랐을까?" |
| 내지에 메시지 두 개 | "잠실은 5.4배 늘고, 종로는 3.7배 늘었어요" | 두 슬라이드로 분리 |
| 강조 너무 많음 | "{잠실}은 {일식}이 {5.4배} {전월 대비} 늘었어요" | "{잠실}은 일식 매출이 전월 대비 {5.4배} 늘었어요" |
| 광고체 마무리 | "지금 바로 페이히어 가입!" | "자세한 데이터는 페이히어 블로그에서 만나요" |
| 데이터 없는 추상 | "외국인 결제가 늘고 있어요" | "외국인 결제, {2024년 35.6%} 늘었어요" |
