# 카드 뉴스 글쓰기 스타일 가이드

## 1. 핵심 원칙 — "한 슬라이드 = 한 메시지"

카드 뉴스는 정보를 압축하는 매체가 아니라 **시선을 끄는 매체**입니다. 정보 욕심을 부리는 순간 가독성이 무너집니다.

- **1슬라이드 1메시지**. 두 가지를 말하고 싶으면 슬라이드를 늘리세요.
- **1줄 8~14자**(표지) / **10~16자**(내지). 손가락이 화면을 스와이프하는 0.5초 안에 읽혀야 합니다.
- **숫자는 본문에 직접 노출**. "매출이 늘었어요" (X) → "매출이 35.6% 늘었어요" (O)
- **강조 키워드는 `{중괄호}`** — 슬라이드당 1~2개, 더는 금지. (도구가 자동으로 포인트 컬러 적용)

## 2. 어조 — 친근체 + 적정 격식

페이히어의 독자는 **자영업 사장님·매장 운영자**입니다. 친근하되 가볍지 않게.

### 권장
- `~었어요`, `~네요`, `~었을까요?`, `~인가요?`
- `~한다는 점`, `~할 수 있어요`, `~눈에 띄어요`

### 금지
- `~다`, `~한다` 같은 문어체 (X)
- `우리는 ~를 발견했습니다` 같은 영어 직역체 (X)
- `놓치지 마세요!`, `지금 바로 신청!` 같은 광고체 (X)
- 이모지 (X). 느낌표는 슬라이드당 0~1개로 절제.

## 3. 문장 구조

### 표지 카피 패턴

**(A) 후킹 질문형** — 호기심 자극
```
main1: "벚꽃 상권 매출,"           (콤마로 호흡)
main2: "어디가 얼마나 올랐을까?"   (질문)
```

**(B) `.zip` 정보 압축형** — 파일 이름처럼 압축감
```
main1: "고유가 피해지원금"
main2: "핵심 요약.zip"
```

**(C) 데이터 단정형** — 결론 미리 던지기
```
main1: "벚꽃 한 주 만에"
main2: "매출 {2.1배} 뛰었어요"
```

### 내지 카피 패턴

**(A) 맥락 → 결론** — 이유와 결과
```
main1: "벚꽃 나들이가 소비로 이어지며"     (맥락)
main2: "{서울 주요 벚꽃 상권 매출}이 올랐어요!"  (결론 + 강조)
```

**(B) 전환 → 질문** — 다음 슬라이드 유도
```
main1: "가장 큰 증가세를 보인 잠실 · 종로,"  (전환)
main2: "{어떤 업종 매출}이 늘었을까요?"      (질문 + 강조)
```

**(C) 데이터 노출형** — 숫자 본문 박기
```
main1: "{잠실}은 일식 매출이"
main2: "전월 대비 {5.4배} 늘었어요"
```

## 4. 강조 키워드 (`{중괄호}`)

`{}`로 감싼 텍스트는 도구에서 **포인트 컬러(주황·코랄·페이히어 시그니처)**로 자동 표시됩니다.

### 강조 후보
- **숫자/단위**: `{35.6%}`, `{2.5배}`, `{300만원}`, `{6/15까지}`
- **핵심 키워드**: `{잠실}`, `{고유가 피해지원금}`, `{서울 주요 벚꽃 상권 매출}`
- **질문의 답이 될 단어**: `{어떤 업종}`, `{어디가}`, `{얼마나}`

### 금지
- 1슬라이드 3개 이상 강조 (시선 분산)
- 조사·접속사만 강조 (`{~는}`, `{그리고}`)
- 전체 문장 강조 (`{전체 메시지 다 강조}`)

## 5. 표현 라이브러리

### 결과·결론
- `~올랐어요`, `~뛰었어요`, `~급증했어요`
- `~를 보였어요`, `~로 나타났어요`
- `~눈에 띄어요`, `~흥미로워요`

### 질문·후킹
- `~어떤 ~일까요?`
- `~얼마나 ~을까요?`
- `왜 ~?`, `~의 이유는?`
- `~했다면, ~?`

### 전환·연결
- `특히,`, `한편,`, `흥미로운 건,`
- `더 자세히 보면,`, `그 안에서도,`
- `~한 결과,`, `~을 살펴보니,`

### 데이터 prefix
- `조사 결과,`, `분석에 따르면,`
- `최근 한 달간,`, `전년 동기 대비,`
- `~를 기준으로,`

### 마무리 슬라이드 (선택)
- `자세한 데이터는 {페이히어 블로그}에서 확인하세요`
- `이런 트렌드, 우리 매장에도 적용해 보세요`
- `다음 카드 뉴스에서 또 만나요`

## 6. 길이 체크리스트

작성 후 항상 글자 수 직접 세서 확인:

| 필드 | 권장 | 절대 한계 |
| --- | --- | --- |
| 표지 `pill` | 1~6자 | 8자 |
| 표지 `main1` | 8~14자 | 16자 |
| 표지 `main2` | 8~14자 | 16자 |
| 내지 `main1` | 10~16자 | 18자 |
| 내지 `main2` | 10~16자 | 18자 |

> **참고**: 공백·구두점 포함 글자 수. 한국어 1자 = 영문 1자로 카운트(예: "벚꽃 상권 매출," = 8자).

길면 압축하고, 짧으면 살리세요. 단 한 글자 차이로도 한 줄에 안 들어가서 어색해질 수 있어요.

## 7. 페이히어 도메인 컨텍스트

페이히어 카드 뉴스의 주요 주제:
- **결제 데이터** 기반 트렌드 (외국인 결제, 업종별 매출, 지역별 매출)
- **매장 운영 TIP** (지원금, 세무, 인건비, 인테리어, 마케팅)
- **계절·이슈성** 콘텐츠 (벚꽃, 휴가철, 명절, 최저임금 등)
- **사장님 사례** (성공/실패 케이스)

이런 주제는 사장님이 **"우리 매장에 적용할 수 있는가?"**의 시선으로 보기 때문에:
- 추상보다 **구체** (지역명, 업종명, 숫자 명시)
- 일반론보다 **데이터** (전년 대비, 평균 대비)
- 멘트보다 **인사이트** ("이런 트렌드가 있다" → "그래서 이렇게 해보세요")
