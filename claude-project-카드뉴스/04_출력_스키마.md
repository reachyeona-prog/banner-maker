# 출력 JSON 스키마

## 1. 최종 출력 형식

**항상 단일 JSON 코드블록**을 출력합니다. 설명·서론·맺음말 일절 금지.

\`\`\`json
{
  "title": "string",
  "source": "string",
  "cover": { ... },
  "interiors": [ { ... }, ... ]
}
\`\`\`

## 2. 전체 스키마

```typescript
type CardNewsSet = {
  title: string;        // 내부 제목 (도구에 입력되진 않음, 식별용)
  source: string;       // 원문 URL (없으면 빈 문자열 또는 "직접 입력")
  cover: CoverSlide;    // 표지 1장
  interiors: InteriorSlide[];  // 내지 3~6장
};

type CoverSlide = {
  type: "E1";           // 고정값 — 카드뉴스 표지 슬롯 ID
  pill: string;         // 카테고리 라벨 (1~6자). 예: "NEWS", "트렌드", "매장 운영 TIP"
  main1: string;        // 메인 카피 1줄 (8~14자 권장)
  main2: string;        // 메인 카피 2줄 (8~14자 권장)
};

type InteriorSlide = {
  type: "E2";           // 고정값 — 카드뉴스 내지 슬롯 ID
  main1: string;        // 메인 카피 1줄 (10~16자 권장)
  main2: string;        // 메인 카피 2줄 (10~16자 권장)
};
```

## 3. 필드별 규칙

### `title`
- 카드 뉴스 세트의 내부 제목. **사용자에게만 보이고 도구엔 안 들어감**.
- 원문 핵심을 한 줄로 요약. 8~20자.
- 예: "벚꽃 상권 매출 분석", "고유가 피해지원금 가이드"

### `source`

- 원문 출처 — URL이 본문에 같이 적혀 있으면 그 값, 아니면 `"직접 입력"` 또는 빈 문자열.
- 도구가 카드 뉴스 메타로만 사용. 누락해도 무방.

### `cover.type`
- 반드시 `"E1"`. 다른 값 금지.

### `cover.pill`
- 카테고리 라벨. 짧고 명료.
- **권장 라벨**: `NEWS`, `트렌드`, `데이터`, `매장 운영 TIP`, `정책 가이드`, `사장님 사례`, `시즌 리포트`
- 길이 1~6자. 영문은 대문자.

### `cover.main1`, `cover.main2`
- 표지 메인 카피. **공백·구두점 포함 8~14자**.
- main1 → main2가 자연스러운 한 호흡으로 읽혀야 함.
- 강조 `{}` 사용 가능하나 표지에선 절제 (0~1개).

### `interiors[].type`
- 반드시 `"E2"`. 다른 값 금지.

### `interiors[].main1`, `interiors[].main2`
- 내지 메인 카피. **공백·구두점 포함 10~16자**.
- 슬라이드당 한 메시지.
- 강조 `{}`는 슬라이드당 1~2개 권장.

### `interiors[]` 배열 길이
- **최소 3장, 최대 6장**.
- 권장 4~5장.

## 4. `{중괄호}` 강조 규칙

도구에서 `{}`로 감싼 텍스트는 **자동으로 포인트 컬러(코랄/주황)**로 표시됩니다.

### 강조 대상 (우선순위)
1. **숫자·단위**: `{35.6%}`, `{2.5배}`, `{300만원}`, `{6/15까지}`, `{10,030원}`
2. **핵심 명사구**: `{서울 주요 벚꽃 상권 매출}`, `{잠실}`, `{고유가 피해지원금}`
3. **질문의 답이 될 단어**: `{어떤 업종}`, `{어디가}`, `{얼마나}`

### 강조 금지
- 조사·접속사만: `{이}`, `{의}`, `{그리고}` (X)
- 전체 문장: `{메인 카피 1줄 전체}` (X)
- 슬라이드당 3개 이상

### 강조 예시 (정/오)

**Good** — 숫자 + 명사구 균형
```
"{잠실}은 일식 매출이 전월 대비 {5.4배} 늘었어요"
```

**Good** — 핵심 키워드 한 덩어리
```
"{서울 주요 벚꽃 상권 매출}이 올랐어요!"
```

**Bad** — 강조 4개 (분산)
```
"{잠실}은 {일식}이 {전월 대비} {5.4배} 늘었어요"
```

**Bad** — 조사 강조
```
"잠실{은} 일식 매출이 늘었어요"
```

## 5. 검증 체크리스트 (출력 전 자체 확인)

JSON 출력 직전, 아래 항목을 머릿속으로 한 번 점검:

- [ ] `cover.pill`은 6자 이하?
- [ ] `cover.main1`, `cover.main2`는 각 14자 이하?
- [ ] `interiors`는 3~6개?
- [ ] 각 `interiors[].main1`, `main2`는 16자 이하?
- [ ] 슬라이드마다 `{}` 강조 1~2개?
- [ ] 친근체(`~요`, `~까요`)로 끝나고 `~다`/`~한다` 없음?
- [ ] 광고체(`놓치지 마세요`, `지금 바로`) 없음?
- [ ] 이모지 없음?
- [ ] 숫자 데이터가 본문에 노출?
- [ ] 슬라이드 간 흐름이 끊기지 않음?

전부 OK면 출력.

## 6. 출력 예 (완성형)

\`\`\`json
{
  "title": "벚꽃 상권 매출 분석",
  "source": "https://payhere.in/blog/cherry-blossom-trends/",
  "cover": {
    "type": "E1",
    "pill": "NEWS",
    "main1": "벚꽃 상권 매출,",
    "main2": "어디가 얼마나 올랐을까?"
  },
  "interiors": [
    {
      "type": "E2",
      "main1": "벚꽃 나들이가 소비로 이어지며",
      "main2": "{서울 주요 벚꽃 상권 매출}이 올랐어요!"
    },
    {
      "type": "E2",
      "main1": "가장 큰 증가세를 보인 잠실 · 종로,",
      "main2": "{어떤 업종 매출}이 늘었을까요?"
    },
    {
      "type": "E2",
      "main1": "{잠실}은 일식·주점·양식이",
      "main2": "최대 {5.4배}까지 뛰었어요"
    },
    {
      "type": "E2",
      "main1": "{종로}는 선물·기념품 매출이",
      "main2": "전월 대비 {61%} 늘었어요"
    },
    {
      "type": "E2",
      "main1": "벚꽃이 진 뒤에도",
      "main2": "이 트렌드, 우리 매장에 적용해 보세요"
    }
  ]
}
\`\`\`

이 출력 형식만 따르면 됩니다. 다른 텍스트는 출력하지 마세요.

## 7. 사용자 추가 요청 처리

사용자가 후속 요청을 할 수 있습니다:

- **"내지 더 추가해줘"** → 기존 JSON에서 `interiors` 배열에 슬라이드 추가하여 전체 JSON 재출력.
- **"표지 톤 바꿔줘 / 더 후킹 있게"** → `cover` 부분만 재작성. 전체 JSON 재출력.
- **"강조 줄여줘"** → `{}` 개수 줄여서 전체 JSON 재출력.
- **"숫자 더 살려줘"** → 본문에 데이터 더 노출하도록 재작성.

**항상 전체 JSON으로 출력**합니다. 부분 출력 / 차이만 출력 금지.
